CSS Specificity

When two or more rules apply to the same element and set the same property, the one with higher specificity wins. Order: inline > ID > class > element.

Element vs class vs ID

Same property (background) set by different selectors. The more specific rule wins.

Live demo

Styled by p (element) – lowest.

Styled by .by-class – overrides element.

Styled by #by-id – overrides class.

CSS (order in file doesn’t matter; specificity does) 
/* Element – specificity 1 */
p {
  background: #e5e7eb;
  padding: 0.5rem;
}

/* Class – specificity 10 */
.by-class {
  background: #bfdbfe;
  color: #1e40af;
}

/* ID – specificity 100 */
#by-id {
  background: #a7f3d0;
  color: #065f46;
  border: 2px solid #059669;
}

Inline style (highest)

Styles in the style attribute beat ID, class, and element selectors (unless the rule uses !important).

Live demo

This has style="background: #fecaca; ..." – inline wins.

HTML (inline style) 
<p style="background: #fecaca; padding: 0.5rem;">
  This has inline style – inline wins.
</p>

Specificity “score” (concept)

Think of specificity as a score. When both rules apply, the higher score wins.

  • Inline style: 1000
  • ID: 100 (per ID)
  • Class / attribute / pseudo-class (e.g. :hover): 10 each
  • Element / pseudo-element (e.g. ::before): 1 each
Tip: When two rules target the same property on the same element, the one with higher specificity wins. If they’re equal, the one that appears later in the CSS wins.
Example 
/* 1 element  = 1 */
p { color: blue; }

/* 1 class    = 10 */
.foo { color: red; }

/* 1 class + 1 element = 11 */
p.foo { color: green; }

/* 1 ID      = 100 */
#bar { color: purple; }
