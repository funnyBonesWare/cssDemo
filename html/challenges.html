CSS Challenges

Build these from scratch using only HTML and CSS. Use the concept demos as reference. No frameworks, no copy-paste—write every line yourself.

How it works

  • Pick a challenge and create a new index.html + styles.css in a folder (or your own repo).
  • Use only the concepts covered on this site: selectors, specificity, box model, display, flexbox, grid.
  • Check the “Concepts” and “Acceptance criteria” before you start.
  • Time estimates are rough; take more if you need. Stretch goals are optional.
Challenge 1 Beginner ~15 min

Profile card

Build a single card that shows a name, a short title, and one line of bio. It should look like a small “profile” or “team member” card.

Concepts: Selectors (element, class), Box model (padding, border, margin)
Task:
  • One div or article with class e.g. card.
  • Inside: name (e.g. heading), role/title, one sentence bio.
  • Style the card with padding, a border, and margin so it doesn’t touch the viewport edges. Give it a max-width (e.g. 320px).
Acceptance criteria: Card has visible padding and border; content doesn’t touch the card edges; there’s space around the card (margin).

Stretch: Add a second card and use a class so both share the same styles.

Challenge 2 Beginner ~10 min

Inline & block text

A short “article” where some words are highlighted inline and some blocks (e.g. a quote) are clearly separate.

Concepts: Display (block, inline), Selectors
Task:
  • One or two paragraphs. Inside a paragraph, wrap a few words in span and style them (e.g. background, padding) so they stay inline.
  • Add a block-level “quote” (e.g. blockquote or a div) that appears on its own line with different styling (e.g. border-left, padding).
Acceptance criteria: Highlighted words sit in the line of text; the quote is on its own line and visually distinct.

Use a class for the quote and style it only with that class.

Challenge 3 Beginner ~15 min

Button row

A horizontal row of 3–4 buttons (links styled as buttons) with even spacing.

Concepts: Flexbox (container, justify-content, gap), Box model
Task:
  • A wrapper div with display: flex. Inside, 3–4 a tags styled as buttons (padding, background, border-radius, no underline).
  • Use gap between items and justify-content: center (or space-between / space-evenly).
Acceptance criteria: Buttons sit in one row; there’s visible gap between them; layout uses flexbox (no floats or inline-block hacks).

Add a hover state (e.g. different background or border) using a pseudo-class.

Challenge 4 Intermediate ~25 min

Feature or pricing grid

A 3-column grid (or 2×2) of “feature” or “pricing” cards. Each cell has a title and a short description.

Concepts: Grid (rows, columns, gap), Box model, Selectors
Task:
  • One grid container. Use grid-template-columns: repeat(3, 1fr) (or 2 columns). Use gap.
  • At least 4 grid items. Each item is a small card (heading + 1–2 sentences). Style cards with padding and maybe a border.
Acceptance criteria: Layout is built with CSS Grid; columns are equal width (1fr); cards have consistent spacing and don’t overlap.

On narrow viewports, make it 1 column using a media query (optional).

Challenge 5 Intermediate ~20 min

Simple page layout

A minimal page with a header (full width) and a main content area. Main content centered with a max-width.

Concepts: Flexbox or Grid, Box model, Display
Task:
  • Header: site title or “My site”, full width, with padding and a background (or border-bottom).
  • Main: wrapper with max-width (e.g. 800px) and margin: 0 auto to center. Put a heading and a paragraph inside.
Acceptance criteria: Header spans full width; main content is centered and doesn’t stretch too wide on large screens.

Add a footer full width at the bottom. Use flexbox on body (column) so footer sticks at bottom if content is short (optional).

Challenge 6 Intermediate ~20 min

Selector drill (no HTML changes)

You’re given HTML. Write only CSS to style specific elements. You may not add or change classes/IDs in the HTML.

Concepts: Selectors (descendant, child >, adjacent +, attribute [href])
Task: Use this HTML structure (copy it as-is): 
<div class="post">
  <h2>Title</h2>
  <p>First paragraph.</p>
  <p>Second paragraph.</p>
  <a href="https://example.com">Link</a>
  <a>No href</a>
  <div>
    <p>Nested paragraph.</p>
  </div>
</div>
  • Style only the first paragraph after h2 (e.g. larger font or different color) using the adjacent sibling combinator.
  • Style only the nested paragraph (inside the inner div) differently—use a descendant selector, but make sure the direct p under .post are not affected the same way.
  • Style only the link that has an href (e.g. color, underline); leave the link without href unstyled or gray.
Acceptance criteria: First paragraph after h2 looks different; nested paragraph has its own style; only the link with href is styled as a link.

Use the child combinator .post > p to style direct paragraphs only, and a descendant selector for the nested one.

Challenge 7 Intermediate ~30 min

Named grid layout

A small dashboard-style layout using CSS Grid named areas: header, sidebar, main, footer.

Concepts: Grid (grid-template-areas, grid-area)
Task:
  • One grid container with grid-template-areas. Define something like: "header header", "side main", "footer footer". Two columns: narrow (sidebar) + flexible (main).
  • Four children: assign each grid-area: header, side, main, footer. Add placeholder text so each region is visible.
Acceptance criteria: Layout uses grid-template-areas and grid-area; header and footer span full width; sidebar and main sit side by side.

Give the grid a min-height and make the main area grow (e.g. 1fr row) so the layout fills the viewport.
